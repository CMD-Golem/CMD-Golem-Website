Key

1.20.2 +

Lock your containers and doors to protect your items and houses from other players and Creepers. Shift right click on the Block to lock or unlock it.

Crafting

The Crafting Table will show a Gold Nugget as result, but you will get the correct item after you take it out.

You can repair your Key, if you drop a Gold Ingot with the Key on an Anvil. To get more durability you can also enchant it with Unbreaking.
This function can be disabled in the settings of the data pack.

Locking (general settings)

If a player locks a container or door only this player can unlocked it again. The Key doesn't matter in this process since the lock only remembers the player.

Additionally you can enable a setting, which allows you to use the name of Keys as password. With those Keys all players which use the right password can unlock the lock. To prevent forgotten passwords the last player which locked the container or door owns it and can in this way also unlock it with any key like described before.

If you want to, you can also set the maximum amount of locked containers per player and in total in the world.
Use the follwing command to reset it to unlimited:

/scoreboard players reset #max_player keylock /scoreboard players reset #max_world keylock
Locked containers

After you locked the container, the items in it can't be accessed even by breaking it or placing a hopper underneath it.
This also means, that locked furnaces and similar blocks, won't process the items in it. If you put items in a locked container, via hopper or similar, those items will be dropped on unlock.

Settings Menu:

  • You can enable automatic unlocking of containers. With this enabled, all containers withhin your defined radius will be unlocked if you own the container or if you have the right key with password selected in your hotbar. After you move away, the container will be locked again.
  • You can change the behavoir of locked containers. You can either make locked containers unbreakable or replace them with holograms. Additionally the container can also remain as block and after destruction it will be replaced with a hologram.

Locked doors and fence gates

All locked doors, trapdoors and fence gates are unbreakable and can't be open. But since normale doors need a support block, even locked doors will break if those gets removed.

When you lock a double door, bough door leafs will be locked.

Settings Menu:

  • You can enable automatic opening of the whole double door. This setting works for all doors, regardless of whether they are locked.
  • You can enable automatic opening of doors. With this enabled, all doors three blocks in front of players will be opened. After you move away, the door will close itself again.
  • You can enable automatic unlocking of doors. With this enabled, all doors three blocks in front of players will be unlocked if they own the door or if you have the right key with password selected in your hotbar. After you move away, the door will be locked again.

Important technical stuff

Be aware that other data packs and mods which change the functionality of containers and doors could break the functionality of this pack. In the case of containers, this could lead to loss of items.

Never kill all entities in the world, since the locked status is stored in special entities.

A server admin can unlock someone elses locked blocks with an Admin Key:

/function keylock:craft/admin_key

If you can't unlock a container or door anymore, first try to break it. If it replaces itself you can use an Admin Key to unlock it. If this doesn't work for your container and you need your items back, you should use this command to restore them:

/function keylock:container/repair with block <container position>

You won't be able to use the this command if you lock the container with the missing items with new items in it again. The items itself aren't stored in the special entity, but in storage. The file is located in the data folder of your world and is called command_storage_keylock.dat.

Adjust lockable Blocks

The easiest way to select your lockable blocks is on the download page of the data pack.

If you need to change it after the inital download, you need to unzip the data pack. In the folder data/tags/blocks all the lockable blocks are defined.
